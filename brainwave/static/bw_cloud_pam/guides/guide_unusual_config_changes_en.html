Unusual Configuration Changes

The two columns display on the right the unusual safe access configuration changes, on the left the unusual changes in a global way per Vault.

The goal here is to identify at a glance if unusual changes are related to access configuration errors, to evaluate their impact, to identify the root cause of these changes and possible corrections.

In each column, the top 5 changes are displayed, sorted from most critical to least critical. At the bottom of the list, the link "See All Changes" allows to list the completeness of configuration changes assessed by the solution.

In the right column, new safes that have just been added to the scope of the analysis are not considered as changes. Only changes that are atypical with regard to the history are evaluated.

In the left column, new vault that have just been added to the scope of the analysis are not considered as changes. For example, an increase in the number of organizations or groups with access to CyberArk due to the creation of a new vault.

The detected changes are clickable to allow you to identify the precise root cause for each item and the remediation suggestions available to you.

By clicking on the "burger" icon at the top right, you can configure the detection of changes:

  • Hide the change because it is not relevant to your context.
  • Consider any increase as an unusual change.
  • Consider only unusual variations, i.e. sudden increases as an unusual change.

By default, only unusual variations are considered, except for the number of "Organizations with CyberArk Access" or the number of "Groups with CyberArk Access" for which any increase is considered.

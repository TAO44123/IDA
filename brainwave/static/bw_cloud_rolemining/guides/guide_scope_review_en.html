The indicators on this page point out data quality issues that could affect the completeness of the Role Mining analysis. There are four indicators to monitor:

  • The number of selected organizations with no known manager. This indicator depends on the selection. Organizations without a manager can still be included in the analysis. However, this means that the information needs to be found elsewhere in order to locate the contact person.
  • The number of contractors. This indicator depends on the selection. Contractors form a population that is widely represented, particularly in IT departments. If the Role Mining aims to create roles in the IT population, it is important to have contractors assist so that the roles are representative of the activity in these organizations.
  • The number of orphaned accounts This indicator depends on the selection. An orphaned account will not appear in the pivot table because the table only displays identities, not accounts. For completeness of the information presented in the workshop, it is important to correct orphaned accounts. This can be done by going to the page of an account, for example.
  • The number of undocumented permissions. This indicator depends on the selection. If the permissions are not understandable by the business lines, the workshops may not be productive. Avoid doing a workshop with the business managers if they do not understand the permissions being presented.

It is suggested that these four indicators be interpreted in the context of the project as well as the scope selected. For example, if individuals who are not attached to an organization are contractors and the objective is to make roles for employees, this will have no impact. However, it is advisable to monitor these five indicators and have them as close to 0 as possible.

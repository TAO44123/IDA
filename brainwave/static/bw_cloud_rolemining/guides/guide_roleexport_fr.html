Le résultat issu des sessions de role mining peut être exporté dans un format Excel. Le fichier peut être utilisé pour alimenter un outil de gestion des identités (IAM) ou pour être exploité par des formulaires de demande d'accès. Le fichier contient plusieurs onglets qui décrivent :

  • les rôles: Il s'agit de la liste des rôles avec leurs caractéristiques, le propriétaire ainsi que l'organisation de rattachement.
  • les identités: Pour chaque rôle, la liste des identités qui devraient avoir ce rôle.
  • les permissions: Pour chaque rôle, la liste des permissions octroyées par ce rôle.
  • les annotations. Les descriptions ou commentaires associés à des permissions.

N'hésitez pas à exporter assez tôt dans le projet les premiers rôles créés afin de valider que les informations produites sont compatibles avec le système qui va les accueillir. Le point important à surveiller est la granularité des permissions. Il faut notamment s'assurer que les codes de permissions en sortie du Role Mining sont connus du système cible ou s'il existe un moyen de les faire correspondre avec les codes connus du système cible.

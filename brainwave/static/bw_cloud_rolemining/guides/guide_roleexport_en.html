The result of the role mining sessions can be exported to an Excel format. The file can be used to feed an Identity Management Tool (IAM) or to be used by access request forms. The file contains several tabs that describe :

  • roles. This is the list of roles with their characteristics, the owner and the organization.
  • identities. For each role, the list of identities that should have this role.
  • permissions. For each role, the list of permissions granted by that role.
  • annotations. Descriptions or comments on permissions.

Don't hesitate to export the first roles created early in the project in order to validate that the information produced is compatible with the system that will host them. The important point to monitor is the granularity of permissions. In particular, it is necessary to make sure that the permissions codes at the output of the Role Mining are known to the target system or if there is a way to match them with the codes. known to the target system.
